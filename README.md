# Пошук у ширину (Breadth-First Search)

## 1. Основи графів: вершини, ребра, шляхи

Граф - це структура даних, яка складається з вершин і ребер.

**Вершина** - це окремий об'єкт графа. Наприклад, у карті міста вершинами можуть бути зупинки, перехрестя або станції метро.

**Ребро** - це зв'язок між двома вершинами. Якщо між двома вершинами є ребро, то з однієї вершини можна перейти до іншої.

**Шлях** - це послідовність вершин і ребер, якою можна дістатися від однієї вершини до іншої.

Наприклад, якщо є такий шлях:

```text
A -> B -> C -> D
```

це означає, що з вершини `A` можна перейти до `D` через вершини `B` і `C`.

Графи бувають:

- **орієнтовані** - ребра мають напрям;
- **неорієнтовані** - ребрами можна рухатися в обидва боки;
- **зважені** - ребра мають вагу, наприклад відстань або час;
- **незважені** - усі ребра вважаються однаковими.

## 2. Представлення графів у пам'яті

У програмуванні граф можна зберігати різними способами. Найчастіше використовують список суміжності або матрицю суміжності.

### Список суміжності

Список суміжності показує, які вершини є сусідами для кожної вершини.

Приклад на Python:

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D", "E"],
    "C": ["A", "F"],
    "D": ["B"],
    "E": ["B", "F"],
    "F": ["C", "E"],
}
```

Тут вершина `A` має сусідів `B` і `C`, тому з `A` можна перейти до цих вершин.

Список суміжності зручно використовувати, коли в графі не дуже багато ребер.

### Матриця суміжності

Матриця суміжності - це таблиця, у якій показано, чи є ребро між вершинами.

Приклад:

```text
    A B C D
A [0 1 1 0]
B [1 0 0 1]
C [1 0 0 0]
D [0 1 0 0]
```

`1` означає, що ребро між вершинами є.

`0` означає, що ребра немає.

Матриця суміжності зручна для швидкої перевірки, чи існує ребро між двома вершинами.

## 3. BFS: алгоритм і його реалізація

BFS, або пошук у ширину, - це алгоритм обходу графа. Він починає з вибраної вершини, потім перевіряє всіх її сусідів, після цього переходить до сусідів наступного рівня.

Алгоритм працює за принципом черги:

```text
першим прийшов - першим обробляється
```

Основні кроки BFS:

1. Вибрати початкову вершину.
2. Додати її в чергу.
3. Позначити її як відвідану.
4. Поки черга не порожня:
   - взяти першу вершину з черги;
   - переглянути всіх її сусідів;
   - якщо сусід ще не був відвіданий, додати його в чергу.

Реалізація BFS на Python:

```python
from collections import deque


def bfs(graph, start):
    visited = set()
    queue = deque([start])
    order = []

    visited.add(start)

    while queue:
        vertex = queue.popleft()
        order.append(vertex)

        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

    return order
```

Приклад використання:

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D", "E"],
    "C": ["A", "F"],
    "D": ["B"],
    "E": ["B", "F"],
    "F": ["C", "E"],
}

print(bfs(graph, "A"))
```

Результат:

```text
['A', 'B', 'C', 'D', 'E', 'F']
```

BFS також можна використовувати для пошуку найкоротшого шляху в незваженому графі.

Приклад:

```python
from collections import deque


def shortest_path_bfs(graph, start, goal):
    visited = {start}
    queue = deque([(start, [start])])

    while queue:
        vertex, path = queue.popleft()

        if vertex == goal:
            return path

        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))

    return None
```

Якщо потрібно знайти шлях від `A` до `F`, результат може бути таким:

```text
A -> C -> F
```

## 4. Приклади використання BFS у житті

BFS використовується не тільки в навчальних задачах, а й у реальних системах.

Приклади:

- **Маршрути в метро або транспорті** - можна знайти найкоротший шлях між станціями або зупинками.
- **Соціальні мережі** - можна знайти людей на певному рівні знайомства, наприклад друзів друзів.
- **Ігри** - BFS допомагає персонажу знайти шлях по карті або лабіринту.
- **Файлова система** - можна обходити папки рівень за рівнем.
- **Комп'ютерні мережі** - можна перевірити, чи є шлях між двома пристроями.
- **Головоломки** - BFS допомагає знайти розв'язок з мінімальною кількістю ходів.

Головна перевага BFS у тому, що він досліджує граф поступово: спочатку найближчі вершини, а потім дальші. Саме тому він добре підходить для пошуку найкоротшого шляху, якщо всі ребра мають однакову вагу.
